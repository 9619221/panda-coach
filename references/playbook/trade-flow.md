## Step 1.5 · 市场状态前置过滤（读 system_v3）

**读** [`trading-system/system_v3.md`](../../trading-system/system_v3.md)（本 skill 内置，无需外部路径）。

计算日线 ATR(14) / 现价：
- ATR/价 < 2% → **震荡市**：禁用突破/破位 60 分制，切 4B 高抛低吸规则（接近区间上沿试空、下沿试多、目标中位、仓位减半）
- 日线实体 > ATR×2 的破位/突破 → **趋势市**：启用 60 分制
- 之间 → **中性**：仓位减半

→ 本结论写进输出模板"道"段，并贯穿全文，不允许小周期信号翻掉市场状态判定。

## Step 2.5 · 庄家思维位（08 庄家思维篇）

**熊猫原话**：「要想想它背后的力量，去想象线背后的故事。」

每次分析必须显式列出以下四类位：

| 类别 | 说明 | 来源 |
|------|------|------|
| **上方流动性池** | 前高 / 24h 高 / BB 上沿 / EMA 上隧道 / 整数关口 | 主力扫多头止损的目标区 |
| **下方流动性池** | 前低 / 24h 低 / BB 下沿 / EMA 下隧道 / 整数关口 | 主力扫空头止损的目标区 |
| **扫单高风险区** | 24h 极值 ± 0.3%（插针区）| 禁止在此挂止损 |
| **主力痕迹** | Spring / 孤立支点 / 敲山震虎 / 试盘洗盘（参 `systems/system-庄家思维.md`） | 若近 24 根 K 线出现，标注并影响方向 |

→ 本段写进输出模板"道"或独立「🎯 庄家思维位」小节，不能省略。

## Step 3 · 60 分进场制打分

**每项满分 20，总分 < 60 直接拒绝开仓。**

### ① 趋势一致性（20 分）

| 共振度 | 得分 | 判定 |
|--------|------|------|
| 4/4 周期（日 / 4H / 1H / 15m）同向 | **20** | 满分 |
| 3/4 同向 | 15 | 过 |
| 2/4 同向 | 10 | 边缘 |
| < 2/4 同向 | **0** | **禁止开仓** |

**判断方法**：收盘价相对 EMA21 / EMA55 位置 + 最近 20 根 K 线的高低点结构。

**必须输出多周期共振表**（不允许只给总分）：

| 周期 | 方向 | 证据 | 得分 |
|------|------|------|------|
| 日线 | 多/空/中 | 价 vs EMA200 + 高低点结构 | — |
| 4H | 多/空/中 | 价 vs EMA50 + 最近 20 根结构 | — |
| 1H | 多/空/中 | 价 vs EMA21 + MACD/RSI | — |
| 15m | 多/空/中 | 价 vs EMA21 + 短期结构 | — |
| **共振** | **X/4** | — | **对应分数** |

### ② 关键位（20 分）

**做多**：入场价上方 ≥ 2× 止损距离内无强压力位 → 20
**做空**：入场价下方 ≥ 2× 止损距离内无强支撑位 → 20
**入场价距关键位 < 止损距离** → **0（位置错了）**

**熊猫原话**：「支撑位不做空，压力位不做多」

### ③ 信号质量（20 分）

**主信号**（必须命中至少一类，否则**封顶 5 分**）—— 以下任一类，每命中一种 +5（上限 20）：

**A. 12 金 K**（权威：`systems/system-12金K.md`，含精讲 12 金 K 补强）
- 警示类（十字星 / 三星）
- 反转类（垂形 / 上吊 / 射击 / 倒锤 / 吞没 / 启明 / 黄昏） —— 合称 **pinbar**
- 进攻类（仙人指路 / 多方炮空方炮 / 三兵三压 / 上升下跌三法 / 一阳指断头鹰）
- ⚠️ 进攻金 K 的「**80% 假突破 / 2B 法则**」：出现在压力 / 支撑位**不要马上跟**，等下一根反向确认

**B. 价格行为信号**（权威：`systems/system-价格行为.md`）
- 简单回调入场（FB / SB 结构） / 复杂回调结束 / RB 结构 / 突破真假识别
- 详见 `concepts/简单回调.md`、`复杂回调.md`、`FB-SB结构.md`、`RB结构.md`、`突破真假.md`

**C. 指标确认**（权威：`systems/system-MACD.md`、`system-布林带.md`）
- MACD 顶底背离 / 金死叉 + 柱状放量
- 布林带冲破通道 / 形态（口袋 / 喇叭口）
- 详见 `concepts/MACD-*.md`（13 张）、`布林带-*.md`（5 张）

**D. 均线 / 趋势线**（权威：`systems/system-均线双龙.md`、`system-Vegas-2B.md`）
- Vegas 隧道关键位 / 双龙战法 / 2B 法则反转 / 123 法则
- 详见 `concepts/Vegas隧道.md`、`2B法则.md`、`一二三法则.md`、`格兰维尔八大法则.md`

**辅助信号（加分项，可选）**：
- 形态确认（`system-形态.md` 12 张）：头肩 / 双重 / 三重 / V 形 / 圆弧 / 菱形 / 三角 / 矩形 / 楔形 / 旗形 / 扩散
- 庄家痕迹（`system-庄家思维.md`）：Spring / 孤立支点 / 敲山震虎 / 试盘洗盘
- 谐波（`法-谐波.md`，10 张）：AB=CD / 加特利 / 蝙蝠 / 蝴蝶 / 螃蟹 / 鲨鱼 / 赛福 / 5-0 / 三重推进
- 缠论（`法-缠论.md`，11 张）：中枢 / 背驰 / 1 / 2 / 3 类买卖点
- SMC 订单流（`法-SMC订单流.md`，8 张）：流动性扫荡 / OB / FVG / PO3 / BMS-SMS

⚠️ **辅助武器不能单独构成入场理由**（"价格行为是正室，其他是妃子"），只用作主信号的强化或冲突预警。

### 加强武器 tie-breaker（边缘分数专用）

**触发条件**：主打分处于 58-62 边缘带。

此时必须跑一遍辅助武器（谐波 / 缠论 / SMC / 波浪），按以下规则调整总分：
- 辅助武器**确认主方向**（如谐波完成形态 / 缠论第 2-3 类买卖点 / SMC 流动性扫荡+OB 回踩 / 波浪 3 浪主升）→ **+5 分**
- 辅助武器**与主方向冲突**（如缠论背驰但你要追多 / SMC 扫多头后无反转就反手）→ **-5 分 并警告**
- 辅助武器**无信号** → **不加不减**

**硬规**：辅助武器独立入场时仓位打 5 折（妃子不能僭越正室）。谐波单用两发子弹规则（轻仓试 + 确认追），见 `methodology/术-仓位管理.md` §10。

### ④ 交易计划完整性（20 分）

缺一减 5：
- 明确方向（多 / 空）
- 明确止损价
- 明确止盈价（且 R:R ≥ 1:1.5）
- 明确保证金 & 杠杆 & 张数

## Step 4 · 算止损止盈

**权威依据**：`methodology/术-止损止盈.md`（保二争五 / 三线止盈 / 1:1.5 硬规）

### 止损规则（硬规）

取以下二者**较近**的一个（更严格）：
1. **关键位法**：最近一个有效支撑 / 压力位 ± 0.2%（详见 `methodology/法-关键位判定.md`）
2. **ATR 法**：入场价 ± (ATR(1H) × 1.5)

- **做多**：止损 = min(关键支撑 × 0.998, 入场 - 1.5×ATR)
- **做空**：止损 = max(关键压力 × 1.002, 入场 + 1.5×ATR)

### 止盈规则

- 必须满足 **R:R ≥ 1:1.5**（熊猫硬规）
- **优先三线止盈**（保二争五）：
  - TP1 = 入场 ± 1R 距离 → 平 50% 仓 + 止损移至入场（保本）
  - TP2 = 入场 ± 2R 距离 → 平 30% 仓
  - TP3 = 入场 ± 5R 距离 → 剩余 20% 仓 + 移动止损
- 单线止盈兜底：取**下一个关键位**；若关键位 R:R < 1:1.5 → 拒绝此单或重选入场
- 详细分批策略见 `methodology/术-止损止盈.md` §6 三线止盈

### 张数计算（永续合约）

```
合约乘数 = 取自 market_get_instruments 的 ctVal
张数 = floor(保证金 × 杠杆 / (入场价 × 合约乘数))
最大张数 = floor(账户可用 × 0.2 × 杠杆 / (入场价 × 合约乘数))
```

### 仓位硬规（权威：`methodology/术-仓位管理.md`）

- 单笔保证金 **≤ 账户可用 20%**（超过 → 警告）
- 单笔风险 **≤ 账户 1-2%**（亏损金额，不是保证金比例）
- 同方向最多 3 笔（避免变相加杠杆）
- 辅助武器（谐波 / 缠论 / SMC）独立入场 → **仓位打 5 折**（"妃子不能僭越正室"）
- 谐波**两发子弹规则**（轻仓试 + 确认追）见 `术-仓位管理.md` §10

## Step 6 · 等确认 → 调 API

**硬规**：输出 7 行后**必须停下来**，等用户明确回复：

| 用户回复 | 动作 |
|---------|------|
| ✅ 「下」/「确认」/「执行」/「go」 | 调 `swap_place_algo_order`（带 tpTriggerPx / slTriggerPx） |
| ⚠️ 「改 X」 | 根据修改重跑 Step 4-5 |
| ❌ 「算了」/「不下」/任何迟疑 | 停在报告阶段 |

### 下单工具调用参数模板（永续）

```json
{
  "instId": "BTC-USDT-SWAP",
  "tdMode": "isolated",
  "side": "buy",
  "posSide": "long",
  "ordType": "limit",
  "px": "74300",
  "sz": "3",
  "tpTriggerPx": "75850",
  "tpOrdPx": "-1",
  "slTriggerPx": "73400",
  "slOrdPx": "-1"
}
```
- `side`：做多 = buy，做空 = sell
- `posSide`：与方向对应（long / short）
- `tpOrdPx`/`slOrdPx` = -1 表示市价止盈止损

### 下单后动作

1. 调 `swap_get_orders`（`ordState=live`）确认挂单成功
2. 记录订单 ID
3. 告诉用户成交情况（未成交 → 挂单中；已成交 → 显示成交价）

## 10 条硬规汇总（违反立即拒绝）

| # | 硬规 | 违反处理 |
|---|------|---------|
| 1 | R:R < 1:1.5 | 拒绝，重算止盈或换入场价 |
| 2 | 止损距关键位 < 0.2% | 拒绝，止损太紧会被插针 |
| 3 | 无止损 | 拒绝，不允许裸单 |
| 4 | 保证金 > 账户可用 20% | 警告，建议降到 20% 以下 |
| 5 | 杠杆 > 10x | 警告"高速 120 码，配不配得上" |
| 6 | 60 分评分 < 60 | 拒绝开仓 |
| 7 | 多周期趋势 < 2/4 一致 | 拒绝，方向不明 |
| 8 | 用户没说"下 / 确认 / 执行" | 停在报告阶段，不调 API |
| 9 | 盘口极端资金费率（\|rate\| > 0.1%） | 警告 |
| 10 | 当前已有同币种持仓 | 警告，避免无意义加仓 |
| 11 | 改已有单的止损（扛单行为） | 拒绝 |
