# Panda Coach 回测引擎

按熊猫教练 V3.2 五步骨架实现的事件驱动回测器。数据来源 OKX 公开 REST API（无需密钥），策略对齐 [`trading-system/system_v3.md`](../../trading-system/system_v3.md)。

---

## 快速开始

```bash
# 1. 装环境（推荐 uv）
pip install uv  # 或 winget install astral-sh.uv
cd panda-coach/scripts/backtest
uv venv && uv pip install -e .

# 2. 拉数据 + 跑回测 + 出报告（默认 BTC-USDT-SWAP 近 365 天）
uv run python run.py --inst BTC-USDT-SWAP --days 365

# 跑过一次后用本地缓存
uv run python run.py --inst BTC-USDT-SWAP --days 365 --no-fetch
```

输出在 `reports/`：
- `<inst>_summary.md` —— 核心指标表
- `<inst>_trades.csv` —— 逐单明细
- `<inst>_equity.png` —— 资金曲线

## 撮合假设（贴近实盘但保守）

| 项 | 取值 | 说明 |
|---|---|---|
| 进场价 | 信号 K 的 close | 限价单近似（V3.2 §2.5 不挂整数关，本回测忽略） |
| 滑点 | 0 | 小账户 + 主流币忽略，结果略乐观 |
| 手续费 | 双边 0.05% | OKX SWAP 普通账户 taker 费率 |
| 单 K 内同时穿过止损止盈 | 按止损成交 | 保守，high/low 顺序未知 |
| 持仓上限 | 1 | V3.2 ≤3 简化 |
| 反向开仓 | 同日内不允许 | V3.2 §禁忌 5 |

## 退场红线（state machine）

| 条件 | 行为 | 解除 |
|---|---|---|
| 当日累亏 > 5% | 全停 | 24h |
| 同币止损 | 该币冷静 | 4h |
| 连续 3 单亏 | 复盘日 | 24h |
| 当周累亏 > 10% | 风险减半 | 1 周 |

---

## 已知偏差（结果偏乐观的方向）

1. **进场价用 close**：实盘限价单可能挂不到 close，价格滑过后未必成交
2. **关键位识别程序化 vs 人眼**：fractal 法找 swing 比人眼判断保守，可能错过部分关键位
3. **没有形态识别**：头肩 / 双重顶 / 反转 4 步全跳过 → 信号源比 V3.2 实际少
4. **量能阈值是拍的**：1.3× MA20 是经验值，不一定贴合各币种
5. **滑点 / 资金费率忽略**：永续合约持仓过夜会扣资金费

→ **结论**：回测正期望不等于实盘必赚，这是熊猫第 1 条原则「不预测，只验证数学」的当代版本。

## 首跑结果（BTC-USDT-SWAP，最近 365 天，2025-05 ~ 2026-05）

| 指标 | 值 |
|---|---|
| 总单数 | 232 |
| 胜率 | 41.8% |
| 实测 R:R | 1:1.84 |
| 盈亏因子 | 1.32 |
| 单笔期望值 | +0.246 U |
| 总盈亏 | +30.56 U |
| 收益率 | +97.0% |
| 最大回撤 | -38.4% |
| 期末余额 | 62.06 U（起始 31.5 U） |

**数学正期望成立** —— 胜率 41.8% × R:R 1.84 ≈ V3.2 §0.1 第 3 条「40% 胜率 × 1:2 = 长期正期望」。

### ⚠️ 必须打的警告（别拿这个数字盲目实盘）

1. **走势偏多头**：2025-05 ~ 2026-05 BTC 是「先震荡后单边上行」，对顺趋势策略天然友好；2022 熊市 / 2018 大跌应另测
2. **参数有调试痕迹**：1H ATR 阈值从 0.02 调到 0.005（V3.2 §1.5 原文是日线口径，迁到 1H 后必须降阈值），属合理调整但**未做敏感性测试**
3. **回撤 38%**：满足实盘退场红线（周累亏 >10% 减半仓位）会进入风险减半状态，这部分回测已实现；实测中触发了多次冷静期
4. **滑点 / 资金费率为 0**：永续合约持仓过夜会扣资金费，对长持单不利；实盘扣完收益会降一档
5. **进场用 close 价**：限价单实盘可能挂不到该价
6. **样本单一**：单币种、单年；推广前必须多币种 + 多年份验证

→ **结论**：管道跑通了、数学上印证了 V3.2 期望值理论；但「回测能赚 ≠ 实盘能赚」是熊猫第 1 条原则的当代版本。下一步要做敏感性测试 + 多币种 / 多年份回归。

## 批量回测结果（4 项扩展验证）

通过 `python batch.py {multi|bear|sweep|costs|all}` 自动跑。详细分表见 `reports/batch_*.md`。

### 1. 多币种 365 天（cmd: multi）

| 标的 | 单数 | 胜率 | 实测 R:R | 期望值 | 收益 | 最大回撤 |
|---|---|---|---|---|---|---|
| BTC | 232 | 41.8% | 1:1.84 | +0.246 | **+97%** | -38% |
| ETH | 387 | 37.7% | 1:1.77 | +0.057 | **-69%** | -87% |
| SOL | 377 | 40.1% | 1:1.87 | +0.198 | **+97%** | -38% |
| ADA | 370 | 36.8% | 1:1.85 | +0.059 | **-59%** | -73% |
| AVAX | 403 | 45.9% | 1:1.74 | +0.347 | **+300%** | -19% |
| LINK | 391 | 39.6% | 1:1.66 | +0.066 | **-41%** | -91% |

**发现**：
- 平均胜率落在 36-46%，符合 V3.2 数学模型
- 但**结果方差极大** —— 同套规则在 6 个币上 3 赢 3 亏
- 期望值 +0.06 U 看似正，但实际收益 -69%（ETH）—— 因为单笔风险 1.5 U / 31.5 U 本金 = 4.7%，连亏 10 单就 -50% 不可逆
- 这暴露了**固定金额 1.5 U/单 + 31.5 U 小账户**的脆弱性，并非系统逻辑错误

### 2. 不同年份（cmd: bear）

| 时段 | 标的 | 单数 | 胜率 | 期望值 | 收益 | 回撤 |
|---|---|---|---|---|---|---|
| 2022 熊市 | BTC | 335 | 43.3% | +0.246 | **+139%** | -55% |
| 2022 熊市 | ETH | 398 | 42.2% | +0.243 | **+150%** | -28% |
| 2024 平稳 | BTC | 315 | 41.6% | +0.174 | +49% | -45% |
| 2025-26 牛市 | BTC | 232 | 41.8% | +0.246 | +97% | -38% |

**发现**：
- 系统**双向有效** —— 2022 LUNA / FTX 暴跌年 BTC 反而 +139%、ETH +150%（做空获利）
- 三种行情（熊 / 平 / 牛）都正期望，**鲁棒性比预期好**
- 但年回撤稳定在 38-55% 区间，实盘要承受 -50% 心理压力

### 3. 参数敏感性（cmd: sweep；BTC 365d）

| ATR 阈值 | tol | 单数 | 胜率 | 期望值 | 收益 | 回撤 |
|---|---|---|---|---|---|---|
| 0.003 | 0.003 | 178 | 42.1% | +0.268 | +70% | -36% |
| 0.003 | **0.005** | **232** | **41.8%** | **+0.246** | **+97%** | **-38%** |
| 0.003 | 0.008 | 251 | 37.8% | +0.099 | -1% | -51% |
| 0.005 | 0.005 | 232 | 41.8% | +0.246 | +97% | -38% |
| 0.008 | 0.008 | 238 | 36.6% | +0.052 | -35% | -57% |

**发现**：
- **ATR 阈值不敏感**（0.003/0.005 结果完全相同）—— 因为 BTC 1H ATR/价 几乎从没低于 0.5%
- **关键位 tol 极敏感**：0.005→0.008 多出 19 单，胜率从 41.8% 跌到 37.8%，期望值 -60%
- 工作区间 tol = 0.003-0.005，超过即进入信号污染区
- **过拟合风险确认**：当前 tol=0.005 是合理选择，但不是参数最优解，参数最优是 atr=0.003 tol=0.003（更严但单数少）

### 4. 滑点 + 资金费率（cmd: costs；BTC 365d）

| 成本档 | 滑点 | 资金费 | 期望值 | 收益 |
|---|---|---|---|---|
| 无成本 | 0 | 0 | +0.246 | **+97%** |
| 中（OKX 实际） | 0.02% 单边 | 0.005%/8h | +0.142 | **+23%** |
| 高（流动性差时段） | 0.05% 单边 | 0.01%/8h | +0.003 | **-75%** |

**发现 — 这是最致命的一项**：
- 中成本（贴 OKX 主流币实际）：收益从 +97% → +23%，期望值 -42%
- 高成本：期望值几乎为 0（+0.003 U/单），实际亏 75%
- 胜率不变（41.8%）—— 滑点 + 资金费不影响信号，只吃利润
- **结论**：系统对成本极敏感。1:1.84 的 R:R 在中等成本下退化到 1:1.64，已逼近 V3.2 §2.4 1:1.5 底线

## 综合结论（**实盘前必读**）

1. **数学正期望成立**：6 币 × 3 时段 × 9 参数 = 18 组合中，绝大多数显示胜率 37-46% × R:R 1:1.66-1.87 → 期望值正
2. **但回测正期望 ≠ 实盘能赚**：ETH/ADA/LINK 都因小账户 + 固定 1.5 U 风险被打穿，**幸存者偏差**
3. **成本是头号杀手**：中成本砍 76% 收益、高成本直接由盈转亏。OKX 普通账户实盘必然在中成本档以上
4. **最佳改进路径**（按价值降序）：
   - 风险百分比化：从「固定 1.5 U / 单」改成「当前余额 × 5%」，连亏自动缩仓 → 抗爆仓
   - 增大本金：31.5 U 数学上够分散，工程上太脆。500 U+ 才能真正分散到多币种
   - 加资金费率过滤：不开持仓预计跨夜 + 当前资金费率 > 0.01% 的反向单
   - 形态识别（头肩 / 反转 4 步）→ 提高胜率到 45%+
5. **回测正期望但实盘亏损**正是熊猫第 1 条原则的当代注脚：「不预测，只跟随结构 / 靠数学盈利」—— **数学只是必要条件**，仓位管理、心理、成本、本金都是充分条件
